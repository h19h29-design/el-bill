# 서울교육 전기요금 절감 진단·피크관리 플랫폼

학교 행정실 담당자가 월별 한전 고지서 데이터를 넣어 전기요금, 사용량, 최대수요전력, 요금제 변경 효과, 피크관리 가이드, 내부/한전 제출 문서 초안을 확인하는 시연용 React MVP입니다.

## 실행

```bash
npm install
npm run dev
```

검증:

```bash
npm run typecheck
npm run lint
npm run test
npm run build
npm run test:e2e
```

## 구현 범위

- React + TypeScript + Vite + Tailwind CSS 기반 브라우저 전용 앱
- 자동진단 흐름: 자동진단 시작 → 자료 업로드 → 학교정보 확인 → 요금제 자동 비교 → 피크관리 방안 생성 → 변경신청 패키지 생성
- 자동진단 결과 화면: 현재 요금제, 추천 요금제, 최근 12개월/3년 절감액, 피크 시나리오 반영 결과, 최종 판단, 다음 행동 버튼 제공
- Recharts 대시보드: 월별 전기요금, 사용량, 최대수요전력, 요금 구성
- SheetJS/xlsx 업로드 파서: 연도별 시트 자동 병합, 자동 인식 결과 확인, 수동 컬럼 매핑
- 한전 파워플래너 자료 가져오기: 사용자가 내려받거나 정리한 엑셀/CSV 업로드, 데이터 유형 선택, 컬럼 매핑
- 파워플래너 월별청구요금 HTML `.xls` 내보내기 파일의 연월, 사용전력량, 청구요금 자동 인식
- TanStack Table 기반 고지서 입력 내역 테이블
- React Hook Form + Zod 기반 피크/사용량 시나리오 입력 검증
- 현재 계약종별/수전전압/요금제를 기준으로 학교용 요금제 후보 TOP 3 자동 비교
- 고지서 기반 차액 추정과 요금표 기반 전체 추정을 분리한 확장 가능한 계산 엔진
- 목표/예상 피크 위험도 판정과 하계/동계/오늘 운영안 자동 생성
- 파워플래너 자료가 있으면 피크관리 화면에서 시간대별 사용량 그래프와 최대부하 시간대 위험도 표시
- 변경계획서, 한전 제출 공문, 변경신청서, 붙임 체크리스트, 계산 근거 요약표, 담당자 검토 필요 항목 패키지 생성
- Playwright E2E smoke: 자동진단, 업로드, 피크관리, 문서 ZIP/PDF 다운로드 흐름 검증
- Vercel 정적 배포 설정: `vercel.json`
- localStorage 24시간 TTL 저장 및 만료 데이터 자동 삭제

## 자동진단 흐름

첫 화면의 `전기요금 자동진단 시작` 버튼을 누르면 사용자가 메뉴를 찾아다니지 않고 아래 순서대로 진행할 수 있습니다.

1. 자료 업로드: 한전 고지서 엑셀/CSV 또는 파워플래너 월별·일별·시간대별 자료를 브라우저에서만 읽습니다.
2. 자동 인식 결과 확인: 시트명, 연도, 필수 컬럼, 누락 컬럼, 자동 매핑 신뢰도를 확인하고 분석을 시작합니다.
3. 요금제 자동 비교: 현재 프로필의 계약종별, 수전전압, 현재 요금제를 기준으로 학교용 후보를 절감액 순으로 비교합니다.
4. 피크관리 방안 생성: 본관/별관 EHP, 강당, 급식실, 특별실, 제외 공간 정보를 반영해 운영안을 생성합니다.
5. 변경신청 패키지 생성: 실제 추천 요금제명과 절감액을 반영한 내부 계획서, 한전 공문, 신청서 미리보기, 체크리스트를 생성합니다.

분석 결과에는 공식 청구액 계산기가 아니라 학교 내부 진단용 추정이라는 안내와, 요금제 변경 신청은 원칙적으로 1년에 한 번만 가능하므로 신중히 검토해야 한다는 안내를 계속 표시합니다.

## 첨부파일 반영

- `0. 전기요금-등촌고.xlsx`: 2020~2025 학년도별 전기요금/사용량 구조를 파악해 `A고등학교` 익명 샘플 데이터와 업로드 파서 규칙에 반영
- `예산절감을 위한 전기요금제 변경 계획(안).pdf`: 관련 근거, 목적, 학교 현황, 요금제 비교, 연간/5년 절감액, 추진일정, 행정사항 문서 구조 반영
- `등촌고등학교 전기요금 변경 신청.pdf`: 수신/제목/관련 근거/붙임 목록 구조 반영
- `전기사용계약 변경신청서.hwp`: 앱에서는 자동 입력 가능한 신청서 항목과 수기 확인 필요 항목을 분리해 미리보기로 구현
- `서울교육_전기요금_관리_대시보드.png`: 파란 사이드바, 흰 패널, 요약 카드, 차트/표/문서 카드 밀도와 색상 체계를 UI 레퍼런스로 사용

원본 파일은 repo에 복사하지 않았고, 실제 학교명·고객번호·담당자·주소·연락처는 샘플에서 마스킹했습니다.

## 주의사항

본 앱은 공식 청구액 계산기가 아니라 학교 내부 진단용 추정 도구입니다. 요금제 변경 신청은 원칙적으로 1년에 한 번만 가능하므로 예상 절감액, 피크 시나리오, 향후 사용량 변동을 신중히 검토한 뒤 진행해야 합니다.

한전 파워플래너 자료는 사용자가 직접 내려받거나 정리한 파일만 업로드합니다. MVP에서는 한전 계정 자동 로그인, 크롤링, 비공식 API 호출을 하지 않습니다. 파워플래너 사용 가능 여부는 한전 파워플래너에서 고객번호로 확인해야 하며, 고객번호는 한전 전기요금 청구서의 10자리 숫자입니다.

`xlsx` 패키지는 요구 스택에 따라 사용했지만 npm audit 기준 고위험 취약점이 보고됩니다. 이 MVP에서는 서버 업로드 없이 사용자의 브라우저 안에서만 파일을 읽도록 제한했고, 운영 전 보완 계획은 `docs/upload-security-plan.md`에 정리했습니다.
