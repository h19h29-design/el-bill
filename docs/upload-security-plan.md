# Upload Security Plan

## 현재 MVP 방침

- 한전 자동 로그인, 크롤링, 비공식 API 호출을 하지 않는다.
- 사용자가 직접 내려받은 한전 고지서, 파워플래너 엑셀/CSV만 업로드한다.
- 파일은 서버로 전송하지 않고 브라우저 로컬에서만 파싱한다.
- 시연용 데이터는 localStorage 24시간 TTL로 만료 처리한다.

## xlsx 패키지 리스크

`xlsx`는 MVP 호환성을 위해 유지하지만 npm audit 기준 고위험 취약점이 보고된다. 현재 구조에서는 서버 업로드가 없으므로 영향 범위를 브라우저 세션으로 제한한다.

## 운영 전 보완 계획

1. CSV와 파워플래너 HTML `.xls`는 자체 파서로 우선 처리한다.
2. 일반 `.xlsx` 파싱은 Web Worker 또는 sandboxed iframe으로 분리한다.
3. 파일 크기, 시트 수, 행 수, 셀 문자열 길이 제한을 추가한다.
4. 운영 환경에서는 대체 파서 또는 서버 격리 파서를 검토한다.
5. 원본 파일, 고객번호, 담당자 연락처, 실제 학교명은 로그와 분석 리포트에 저장하지 않는다.
6. 업로드 실패 원인은 사용자 안내로만 표시하고 원본 파일 내용은 외부 전송하지 않는다.

## 검증 기준

- 필수 컬럼 누락 시 오류가 아니라 수동 매핑 안내를 보여준다.
- CSV 한글 헤더는 UTF-8 기준으로 보존하고, 깨짐 감지 시 `euc-kr` 디코딩을 시도한다.
- E2E smoke에서 고지서 `.xls`, 파워플래너 CSV, 문서 ZIP/PDF 다운로드를 검증한다.
